• Zunächst wird das entsprechende Wunsch-Repositorium ins eigene Verzeichnis geforked. Mit git clone wird das Verzeichnis nun auf die Festplatte gespeichert, und zwar ins momentane Arbeitsverzeichnis. Dies kann auch händisch über die Webseite erfolgen.
  • Mit git add können nun Änderungen im Verzeichnis auf der Festplatte gemacht werden. Diese werden nicht zwischengespeichert. Erst mit git commit speichert Änderungen inklusive einer Nachricht, die diese beschreiben sollte. Das allerdings ebenfalls nur lokal.
  • Mit git push können die Änderungen nun vom lokalen ins entfernte Repositorium übertragen werden. Ein vorhergehendes Commit ist dabei obligatorisch.
  • Mit git pull können aktuelle Änderungen nun auf die lokale Version übertragen werden.
